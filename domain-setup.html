Step 01 · ~15 minutes
Get Your Domain — Free Route
ramana.is-a.dev — zero cost, real domain, full DNS control
🆓
Yes, this is genuinely free — no credit card, ever
is-a.dev is an open-source project (5,000+ contributors, backed by Cloudflare) that gives developers a free subdomain like ramana.is-a.dev. You get full DNS control, free HTTPS, and it points to your Vercel deployment exactly like a paid domain would. It's used by thousands of real developer portfolios. Skip Freenom (.tk/.ml/.cf) — that service is unreliable now and has a reputation for revoking domains without warning.

Here's exactly how to claim ramana.is-a.dev in about 15 minutes of actual work, then a 1–2 day wait for approval:

domains/ramana.json — the file you'll create
{ "description": "Ramana Vemunoori — AI Engineer Portfolio", "repo": "https://github.com/Ve0602/ramana-portfolio", "owner": { "username": "Ve0602", "email": "vemunooriramana0602@gmail.com" }, "record": { "CNAME": "cname.vercel-dns.com" } }
The exact steps:
  1. Go to github.com/is-a-dev/register and click "Fork" (top right)
  2. In your forked repo, navigate to the /domains folder
  3. Create a new file named ramana.json
  4. Paste the JSON above (it already has your real GitHub username and email)
  5. Commit the file, then open a Pull Request back to the original repo
  6. A bot checks your PR automatically — usually approved within 24–48 hours
  7. Once merged, ramana.is-a.dev is live and pointed at Vercel
💡 Even faster — ask any AI to walk you through it
Paste this into Claude or any AI assistant: "I want to register a free .is-a.dev subdomain. GitHub username: Ve0602, email: vemunooriramana0602@gmail.com, desired subdomain: ramana, hosting: Vercel CNAME cname.vercel-dns.com. Walk me through forking the repo and submitting the PR." — it'll generate the exact file and guide you click by click.

Vercel is the best free hosting for static portfolios. Your vercel.json is already pre-configured with the right settings. Just connect the repo.

Step 04 · ~5 minutes + wait for PR approval
Connect DNS — ramana.is-a.dev
Point your free domain at Vercel

If you already submitted the domains/ramana.json PR in Step 1, this step is mostly already done — you just need to confirm the CNAME value matches your live Vercel deployment.

✅ How is-a.dev DNS works
Unlike a traditional registrar, you don't touch DNS records yourself — the record.CNAME field in your ramana.json file IS the DNS record. Once your PR merges, is-a.dev's Cloudflare-backed system automatically creates the CNAME pointing ramana.is-a.dev → cname.vercel-dns.com.
Add the domain inside Vercel too
Vercel Dashboard → Your Project (ramana-portfolio) → Settings → Domains → Add Domain → type: ramana.is-a.dev → Vercel detects the CNAME automatically (no further input needed) → Wait for the green "Valid Configuration" checkmark
  • Confirm your PR was merged on github.com/is-a-dev/registerCheck the Pull Requests tab — look for a green merge checkmark
  • Add ramana.is-a.dev in Vercel → Settings → Domains
  • Wait for Vercel's "Valid Configuration" checkmarkUsually instant since the CNAME is already correct
  • Visit ramana.is-a.dev in browserConfirm the green padlock (HTTPS) and your portfolio loads

⚠ Always copy the exact values Vercel's Domains panel shows you — they're auto-generated per project
⚠ Delete conflicting records first
Registrars add default A records when you buy a domain. Delete any existing A records for "@" before adding the Vercel one, or the connection will fail silently.
