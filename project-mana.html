How It Was Built

Full Build Workflow

01
Design
Figma — Complete UX Flow
Designed end-to-end user flows in Figma — cook onboarding, dietary preference setup, menu browsing with AI match scores, subscription checkout, and real-time order tracking. 40+ screens across 4 user types: consumer, cook, admin, delivery.
Figma40+ screens4 user types
02
Frontend
React — Component Architecture
Component-based React frontend with Context API state management. CookCard, MenuBrowser, DietaryFilter, SubscriptionPlan, OrderTracker, and AIMatchScore components. Mobile-first responsive design.
ReactContext APIMobile-first
03
Backend
Spring Boot REST API
Java Spring Boot backend handling cook registration, menu management, order lifecycle (placed → confirmed → preparing → out for delivery → delivered), subscription billing, and AI matching requests. JWT authentication with refresh tokens.
Spring BootJWT AuthREST API
04
AI Engine
Dietary Matching Algorithm
Matching algorithm scores cook menus against user dietary goals using cosine similarity on meal nutritional vectors. Inputs: user goals (weight loss, high protein, vegan, diabetic-friendly) × cook menu nutrition profiles → match %. 0.1s inference.
Cosine SimilarityNutritional VectorsPython
05
Database
MongoDB + SQL Hybrid
MongoDB for flexible cook profiles, dynamic menus, and order documents. SQL (PostgreSQL) for subscription billing cycles, payment records, and user accounts. Redis for session caching and real-time cook availability.
MongoDBPostgreSQLRedis
